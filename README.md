## ‚öôÔ∏è Subnet Incentive Mechanism (Core Logic)

The system relies on a decentralized network of competing validators and miners. Validators ensure the integrity and quality of the reverse prompt outputs, while miners provide the distributed compute required for complex reverse-engineering tasks.

### üîÑ Data Flow: The Reverse Prompt Challenge

1.  **Challenge Issuance:** A **Validator** generates a outputs from a hidden *ground-truth prompt* ($A$) and sends these outputs to the miner network as a challenge.
2.  **Miner Responses:** Multiple **Miners**‚Äîeach running a different RPE model or algorithm‚Äîcompete to guess the original prompt and submit their candidate prompts ($P$, the solution).
3.  **Scoring and Consensus:** The **Validator** tests each candidate prompt by regenerating outputs and scoring how closely they match the originals. The Miner that submits the best-scoring prompt earns **TAO rewards** (or fees from user tool usage).
4.  **Constantly Improving:** This competition incentivizes the rapid adoption and refinement of the most effective RPE algorithms, directly increasing the ROI for users.
