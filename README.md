## Subnet Incentive Mechanism (Core Logic)

The system uses a decentralized network of competing validators and miners. Validators ensure output quality, while miners provide the distributed compute for RPE tasks.

### Data Flow: The Reverse Prompt Challenge

1.  **Challenge Issuance:** A **Validator** generates outputs from a hidden *ground-truth prompt* ($A$) and sends them to the miner network.
2.  **Miner Responses:** Multiple **Miners** (each running a different RPE model) compete to guess the original prompt and submit their candidate prompts ($P$).
3.  **Scoring and Consensus:** The **Validator** tests each candidate prompt by regenerating outputs and scoring how closely they match the originals. The Miner with the best-scoring prompt earns **TAO rewards** (or user fees).
4.  **Constantly Improving:** This competition rapidly refines the most effective RPE algorithms, increasing user ROI.

### Scoring Metrics for Prompt Accuracy

## Subnet Scoring Metric: Design Validation

This metric assesses design quality and effectiveness, rewarding **precision** and **semantic understanding**.

#### 1. Element-Level Accuracy (**60%**)
* **Color Recognition (10%):** Correct identification of primary colors and palettes.
* **Font Recognition (10%):** Accurate identification of font styles.
* **Layout Analysis (10%):** Precise detection of negative space and visual focal points.
* **Semantic Understanding (30%):** Correct interpretation of color palette meaning, font style intent, and visual importance/hierarchy.

#### 2. Overall Design Understanding (**40%**)
* **Style Classification (20%):** Accurate categorization of the overall design aesthetic (e.g., vintage, futuristic).
* **Visual Metaphor Interpretation (20%):** Correctly describing the abstract or metaphorical meaning of the design.

The validator assesses the subnet's ability to accurately identify and interpret both **explicit design attributes** and their **underlying semantic meaning** from design images.
